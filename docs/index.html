Mora — Built with Opus 4.7

An on-device, dyslexia-aware ESL tutor for L1-Japanese kids. Built in 5 days for the Built with Opus 4.7 hackathon (Anthropic × Cerebral Valley, Apr 21–28, 2026).

The problem

My son is eight. He has dyslexia. He's also learning English as a second language — his first language is Japanese, and our family moved from Japan to the US last year.

His school's IEP — the structured plan for dyslexia support — is gated behind finishing ESL. That's a one-year delay before he gets the help he actually needs. Barton tutoring, the gold-standard structured-literacy program, is priced out of reach for a family-funded effort.

For a child user, on-device privacy is non-negotiable: no audio, no transcripts, no per-trial details may leave the iPad.

What it does

Yokai mentor + warmup

Each week, a hand-illustrated yokai mentor introduces the practice arc with a voiced clip. The yokai shell turns daily phonics practice into a quest, not a drill.

Tile-board decoding (multisensory phonics)

The tile board breaks every word into graphemes the learner has explicitly mastered, so what shows up on screen is always decodable, never guessed from context. Multisensory phonics — Orton-Gillingham's core — means moving the letters yourself.

On-device pronunciation feedback

The learner speaks. An INT8-quantized wav2vec2-xlsr-53-espeak-cv-ft phoneme posterior runs in CoreML. The engine catches L1-Japanese substitutions — v as b, l as r, sh as s — and turns them into targeted coaching, not red Xs. No audio leaves the device.

Decodable sentences

Each sentence is generated against the learner's mastered grapheme set. The yokai reads first; the tiles fill as the child speaks.

How it's built

Five local Swift packages with one-way dependencies — MoraCore ← MoraEngines ← MoraUI, with MoraTesting and MoraMLX cross-cutting. The thin Mora/ iOS app target wires SwiftData and presents RootView; all real logic lives in the packages.

  • MoraCore — domain model + SwiftData persistence; L1Profile protocol with JapaneseL1Profile as the v1 implementation.
  • MoraEngines — SessionOrchestrator state machine, AssessmentEngine, SpeechEngine / TTSEngine protocols, CurriculumEngine, ContentProvider, TemplateEngine.
  • MoraUI — SwiftUI only; views observe the orchestrator, never own business logic.
  • MoraTesting — fakes shared across test targets.
  • MoraMLX — CoreML-bundled wav2vec2 phoneme posterior; reserved for future MLX-hosted on-device LLM work.

On-device invariant: no raw audio, transcripts, or per-trial details leave the device. The only planned cloud touchpoint is CloudKit private DB for parent-mode sync — not in the v1 scope.

Built with Claude Code

I'm not a Swift engineer by trade. I built mora in five days with Claude Code and Opus 4.7. Some of the workflow was deliberately structured to push the model:

  • 100+ structured PRs in 5 days — every one backed by a written spec under docs/superpowers/specs/ and an implementation plan under docs/superpowers/plans/. Spec → plan → execute, repeat.
  • Parallel sub-agents on isolated git worktrees. A four-phoneme decodable-sentence content batch that would have taken six hours sequential, finished in one — each phase ran as a separate sub-agent on its own git worktree, no cross-batch state. The default "no parallel implementation agents" guidance from the superpowers skill set applies to single-worktree conflicts; isolation: "worktree" defeats it because each agent gets its own filesystem copy.
  • Custom oslog-stream-to-file skill. Streams Apple OSLog from the running iPad into /tmp/<repo>.log, which Claude reads directly via the Read tool. The classic debug round-trip ("user takes a screenshot of Console.app, pastes it") collapses into a conversation.
  • Spec-driven discipline. Brainstorming → writing-plans → executing-plans, with the user-side review gate at each step. Self-authored skills include pr-autofix-loop (monitor GitHub Actions + Copilot reviews, fix, push, repeat until green) and conduit-archive-to-app-store (CLI orchestration of Product → Archive → Validate → Distribute in parallel for iOS + Mac Catalyst).
